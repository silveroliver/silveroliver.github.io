OECD / CLI ontology portal

Work in progress to design data models for Crop protection models (and hopefully beyond).

Providing a common set of basic information required to identify and exchange data about a product registered in a given teritoty. This should be sufficient for any user (including a machine) to unambigiously search and distinguish between different registered products.

Models

Domain models - Core

  1. Plant protection product model

    An ontology describing a key set of shared properties and hooks for plugging in model extensions.

  2. Packaging model

    A model to describe common forms of reference data or picklist values.

  3. Refence data model

    A model describing the types of location commonly referenced by the Environment Agency.

Domain models - Domain specific

  1. Safety model (AgriGuide)

    A model describing the intended actions by water companies.

  2. PPE model

    A model describing permits of discharge into waterbodies.
